This guide walks you through embedding the Yoyo scripting engine into a C++ project using the YVM backend.

<Steps>
  <Step title="Build Yoyo from source">

Clone the repository and build with Meson:

```bash
git clone https://github.com/git-i/yoyo-lang.git
cd yoyo-lang
meson setup build -Dexecutor=yvm
ninja -C build
```

This produces `libyoyo` in `build/` and places compiled objects under `build/subprojects/`.

  </Step>
  <Step title="Set up your C++ project">

Add Yoyo to your `meson.build` as a subproject, or link manually:

```meson meson.build
project('my-app', 'cpp', default_options: ['cpp_std=c++20'])

yoyo = subproject('yoyo-lang')
yoyo_dep = yoyo.get_variable('yoyo_dep')

executable('my-app', 'main.cpp', dependencies: [yoyo_dep])
```

Or link manually with a compiler:

```bash
g++ -std=c++20 -Iyoyo-lang/include/yoyo -Lbuild main.cpp -lyoyo -lgc -lffi -o my-app
```

  </Step>
  <Step title="Create an engine and app module">

In your C++ code, instantiate `YVMEngine` and create an `AppModule` to expose your C++ functions to Yoyo scripts:

```cpp main.cpp
#include <yvm/yvm_engine.h>
#include <yvm/app_module.h>
#include <iostream>

// A C++ function to expose to Yoyo
int32_t my_print(void* arg) {
    std::string_view sv = Yoyo::Engine::viewString(arg);
    std::cout << sv << "\n";
    return 0;
}

int main() {
    Yoyo::YVMEngine engine;

    // Create an app module named "app"
    auto mod = engine.addAppModule("app");

    // Register the C++ function with a Yoyo type signature
    auto result = mod->addFunction("(x: &str) -> i32", my_print, "print");
    if (result != Yoyo::Result::Success) {
        std::cerr << "Failed to register function\n";
        return 1;
    }
```

  </Step>
  <Step title="Load and compile a Yoyo script">

Add a Yoyo source module, compile it, and prepare for execution:

```cpp main.cpp (continued)
    std::string source = R"(
main: fn = {
    app::print(&"Hello from Yoyo!");
}
)";

    auto yoyo_mod = engine.addModule("main", source);
    if (!engine.compile()) {
        std::cerr << "Compilation failed\n";
        return 1;
    }
    engine.prepareForExecution();
```

  </Step>
  <Step title="Execute the script">

Find the `main` function, create a fiber for it, and run the engine:

```cpp main.cpp (continued)
    // Find and run the main function
    auto fn = engine.findFunction(yoyo_mod, "main::main");
    if (!fn) {
        std::cerr << "Function not found\n";
        return 1;
    }

    auto fiber = engine.createFiber(fn.value());
    engine.execute();

    return 0;
}
```

Running this will print:

```
Hello from Yoyo!
```

  </Step>
</Steps>

## Complete example

Here is the full `main.cpp` assembled:

```cpp main.cpp lines
#include <yvm/yvm_engine.h>
#include <yvm/app_module.h>
#include <iostream>

int32_t my_print(void* arg) {
    std::string_view sv = Yoyo::Engine::viewString(arg);
    std::cout << sv << "\n";
    return 0;
}

int main() {
    Yoyo::YVMEngine engine;

    auto mod = engine.addAppModule("app");
    mod->addFunction("(x: &str) -> i32", my_print, "print");

    std::string source = R"(
main: fn = {
    app::print(&"Hello from Yoyo!");
}
)";

    auto yoyo_mod = engine.addModule("main", source);
    engine.compile();
    engine.prepareForExecution();

    auto fn = engine.findFunction(yoyo_mod, "main::main");
    auto fiber = engine.createFiber(fn.value());
    engine.execute();

    return 0;
}
```

## Registering multiple functions

You can register as many C++ functions as you need before calling `compile()`:

```cpp
auto mod = engine.addAppModule("test");
mod->addFunction("(x: &str) -> i32", print_fn, "print");
mod->addFunction("(x: bool, y: &str) -> void", assert_fn, "assert");
mod->addFunction("(x: u64) -> void", print_int_fn, "print_int");
```

The corresponding Yoyo script calls them as `test::print(...)`, `test::assert(...)`, and `test::print_int(...)`.
