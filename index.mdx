## What is Yoyo?

Yoyo is a statically typed, embeddable scripting language written in C++20. It is designed to be compiled and run directly inside your C++ application, giving you a safe, expressive scripting layer without the overhead or unsafety of traditional FFI approaches.

<Note>
  Yoyo is an active research language. The implementation is still evolving and not yet production-ready.
</Note>

## Key features

## Declaration syntax

All declarations in Yoyo follow a unified syntax inspired by [cppfront](https://github.com/hsutter/cppfront):

```yoyo
name: kind = value;
```

This applies to variables, functions, types, and constants:

```yoyo
// Variable
x: i32 = 42;
name: str = "hello";

// Type-inferred variable
y: mut = 10;

// Function
add: fn (a: i32, b: i32) -> i32 = return a + b;

// Struct
Point: struct = {
    x: f32,
    y: f32
}
```

## Two backends

<Tabs>
  <Tab title="YVM (default)">
    The **Yoyo Virtual Machine** is a custom bytecode interpreter that JIT-compiles via libffi. It is the recommended backend for embedding, offering fast startup and a simpler build configuration.

    ```cpp
    #include <yvm/yvm_engine.h>
    #include <yvm/app_module.h>

    Yoyo::YVMEngine engine;
    auto mod = engine.addAppModule("app");
    mod->addFunction("(x: &str) -> void", my_print, "print");
    engine.addModule("main", source_code);
    engine.compile();
    engine.execute();
    ```
  </Tab>
  <Tab title="LLVM">
    The **LLVM backend** compiles Yoyo directly to native machine code using LLVM IR, providing maximum runtime performance for compute-intensive scripts.

    ```cpp
    #include <llvm/llvm_engine.h>
    #include <llvm/app_module.h>

    Yoyo::LLVMEngine engine;
    auto mod = engine.addAppModule("app");
    engine.addModule("main", source_code);
    engine.compile();
    engine.execute();
    ```
  </Tab>
</Tabs>

## Get started

<Steps>
  <Step title="Install dependencies">
    Build Yoyo from source using Meson. You need a C++20 compiler, the Boehm GC library, and either LLVM or libffi depending on your chosen backend.
  </Step>
  <Step title="Add Yoyo to your project">
    Use Meson's `subproject` or link against the built `libyoyo` library and include `include/yoyo/` in your include path.
  </Step>
  <Step title="Create an engine and expose your API">
    Instantiate `YVMEngine` or `LLVMEngine`, add an `AppModule`, and register any C++ functions your scripts will call.
  </Step>
  <Step title="Load and execute scripts">
    Call `addModule()` with your Yoyo source, then `compile()` and `execute()`.
  </Step>
</Steps>
